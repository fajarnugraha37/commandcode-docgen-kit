# Documentation Experience

Use task-oriented document modes. Each page must have one primary mode: tutorial, how-to, explanation, reference, runbook, decision-record, migration-guide, or troubleshooting. Keep navigation user-journey oriented, define stable aliases, deprecation/replacement metadata, search keywords, related pages, and clear next steps. Do not turn every page into a generic explanation.
