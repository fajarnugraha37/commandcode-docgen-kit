# Reference Writing

Reference pages optimize for lookup and completeness. Prefer structured tables, grouped sections, stable terminology, and explicit source-grounded constraints.

Depending on the subject, capture:

- names, paths, methods, schemas, topics, tables, settings, defaults, and ownership;
- required/optional fields and valid values;
- lifecycle or ordering constraints;
- side effects and failure behavior;
- compatibility or version constraints;
- examples that demonstrate actual repository usage;
- links to conceptual pages explaining why the reference exists.

Do not merely dump identifiers. Organize them around how engineers search for answers.
