# Evidence-Derived Examples

Examples must be derived from schemas, tests, fixtures, contracts, source values, or normalized models. Mark placeholders explicitly. Prefer happy path, alternative branch, validation failure, dependency failure, concurrency conflict, retry/DLQ, and recovery scenarios when evidenced. Never fabricate credentials, identifiers, production endpoints, or business outcomes.
