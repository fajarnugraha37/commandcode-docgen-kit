# Guide Writing

A guide is a task path, not an essay. Include when applicable:

- goal and when to use the guide;
- prerequisites and assumptions;
- exact sequence of actions;
- configuration or code examples grounded in the repository;
- expected result after important steps;
- verification checks;
- common failure modes and recovery;
- rollback or safety notes;
- links to deeper concept/reference pages.

Do not invent commands, endpoints, or configuration. When a required step is not evidenced, mark it as unknown rather than guessing.
