# Guide Writing

A guide is task-oriented. Include:

1. goal and when to use the guide;
2. prerequisites and safety constraints;
3. ordered steps grounded in repository evidence;
4. verification of success;
5. common failure modes and recovery when evidenced;
6. links to deeper concepts/reference.

Never invent commands or operational procedures that cannot be supported from the repository or explicit project instructions.
